Inicialmente, foi escolhida a API PandaScore por fornecer informações sobre equipes de esports, alinhando-se ao tema do projeto. Durante a implementação, verificou-se que a API apresentava restrições de acesso via navegador (CORS) quando utilizada em uma aplicação desenvolvida apenas com HTML, CSS e JavaScript, exigindo a utilização de um backend para intermediar as requisições. Como o projeto previa apenas o desenvolvimento front-end, optou-se pela substituição da API pela FreeToGame, que permite consumo direto pelo navegador e atende aos requisitos da atividade.
